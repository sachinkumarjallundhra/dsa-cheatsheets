🔀 Reorder List – Java Cheat Sheet

📌 What Is It?

The Reorder List problem (LeetCode 143) asks you to reorder a singly linked list in-place from L0 → L1 → … → Ln-1 → Ln to L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → ….

🎯 Solution Pattern (Find Middle, Reverse, Merge)

// Java Solution
class Solution {
    public void reorderList(ListNode head) {
        if (head == null || head.next == null) return;
        // 1. Find the middle
        ListNode slow = head, fast = head;
        while (fast.next != null && fast.next.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }
        // 2. Reverse second half
        ListNode prev = null, curr = slow.next;
        while (curr != null) {
            ListNode nextTemp = curr.next;
            curr.next = prev;
            prev = curr;
            curr = nextTemp;
        }
        slow.next = null;
        // 3. Merge two halves
        ListNode first = head, second = prev;
        while (second != null) {
            ListNode tmp1 = first.next, tmp2 = second.next;
            first.next = second;
            second.next = tmp1;
            first = tmp1;
            second = tmp2;
        }
    }
}

✅ Use Cases

  • Reordering a linked list in a specific pattern
  • Interview questions on pointer manipulation
  • Combining multiple linked list techniques

📝 Algorithm Steps

  1. Use fast and slow pointers to find the middle of the list
  2. Reverse the second half of the list
  3. Merge the two halves alternately

⏱️ Time & Space Complexity

  • Time: O(N) for N nodes
  • Space: O(1) (in-place)

🧪 Example

Input: head = [1,2,3,4]
Output: [1,4,2,3]

Input: head = [1,2,3,4,5]
Output: [1,5,2,4,3]
