üìå What Is It?

The Design HashMap problem (LeetCode 706) asks you to implement a basic HashMap without using any built-in hash table libraries. You need to support put, get, and remove operations.

üéØ Solution Pattern (Separate Chaining)

// Java Solution (Separate Chaining)
class MyHashMap {
    private static class Node {
        int key, value;
        Node next;
        Node(int k, int v) { key = k; value = v; }
    }
    private final int SIZE = 1009; // A prime number for better distribution
    private Node[] buckets = new Node[SIZE];

    public void put(int key, int value) {
        int idx = key % SIZE;
        if (buckets[idx] == null) {
            buckets[idx] = new Node(-1, -1); // Dummy head
        }
        Node prev = find(buckets[idx], key);
        if (prev.next == null) prev.next = new Node(key, value);
        else prev.next.value = value;
    }

    public int get(int key) {
        int idx = key % SIZE;
        if (buckets[idx] == null) return -1;
        Node prev = find(buckets[idx], key);
        return prev.next == null ? -1 : prev.next.value;
    }

    public void remove(int key) {
        int idx = key % SIZE;
        if (buckets[idx] == null) return;
        Node prev = find(buckets[idx], key);
        if (prev.next != null) prev.next = prev.next.next;
    }

    // Find previous node of the node with given key
    private Node find(Node head, int key) {
        Node node = head, prev = null;
        while (node.next != null && node.next.key != key) {
            node = node.next;
        }
        return node;
    }
}

‚úÖ Use Cases

  • Implementing custom hash tables
  • Understanding hash collisions and chaining
  • Learning low-level data structure design

üìù Algorithm Steps

  1. Choose a bucket size (preferably a prime number)
  2. For each operation, compute the bucket index using key % SIZE
  3. Use a linked list (separate chaining) to handle collisions
  4. For put, insert or update the value
  5. For get, search for the key and return its value or -1
  6. For remove, delete the node with the given key if it exists

‚è±Ô∏è Time Complexity

  • Average: O(1) for put, get, remove (with good hash distribution)
  • Worst-case: O(N) if all keys hash to the same bucket

‚ö†Ô∏è Edge Cases to Consider

  • Duplicate keys (should update value)
  • Removing non-existent keys
  • Negative or large keys

üß™ Example

MyHashMap map = new MyHashMap();
map.put(1, 1); // map = {1=1}
map.put(2, 2); // map = {1=1, 2=2}
map.get(1);    // returns 1
map.get(3);    // returns -1 (not found)
map.put(2, 1); // update value, map = {1=1, 2=1}
map.get(2);    // returns 1
map.remove(2); // remove key 2, map = {1=1}
map.get(2);    // returns -1 (not found)
