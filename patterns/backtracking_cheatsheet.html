🔄 Backtracking – Java Cheat Sheet

📌 What Is It?

Backtracking is a technique for solving recursive problems by trying all possible solutions and undoing the last step if the solution is not feasible. It is commonly used for problems involving permutations, combinations, and constraints.

🧱 Pattern Template

void backtrack(List<Integer> current, int[] nums, int start) {
    // Base case: If a solution is found, process it
    if (isSolution(current)) {
        results.add(new ArrayList<>(current));
        return;
    }

    for (int i = start; i < nums.length; i++) {
        // Make a choice
        current.add(nums[i]);

        // Explore further
        backtrack(current, nums, i + 1);

        // Undo the choice
        current.remove(current.size() - 1);
    }
}

✅ Use Cases

  • Generating all subsets, permutations, or combinations
  • Solving constraint satisfaction problems (e.g., N-Queens, Sudoku)
  • Finding paths in a maze or graph
  • Partitioning problems (e.g., Palindrome Partitioning)

💡 Pro Tips

  • Use recursion to explore all possibilities.
  • Always undo the last choice (backtrack) before moving to the next possibility.
  • Prune unnecessary branches early to optimize performance.
  • Edge cases: empty input, single-element input.
