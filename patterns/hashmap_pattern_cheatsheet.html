üìå What Is It?

The HashMap Pattern is a common approach in coding interviews for solving problems that require fast lookups, counting, grouping, or mapping relationships between data. HashMaps (or HashTables) provide average-case O(1) time for insert, delete, and search operations.

üéØ Solution Pattern (Java Example)

// Java Example: Counting frequencies
Map<Integer, Integer> map = new HashMap<>();
for (int num : nums) {
    map.put(num, map.getOrDefault(num, 0) + 1);
}

// Java Example: Checking for existence
if (map.containsKey(target)) {
    // do something
}

‚úÖ Use Cases

  • Counting frequencies of elements
  • Finding duplicates or unique elements
  • Two Sum and similar lookup problems
  • Grouping anagrams or similar items
  • Mapping relationships (parent-child, etc.)

üìù Algorithm Steps

  1. Initialize a HashMap
  2. Iterate through the data and update the map as needed
  3. Use get, put, containsKey, etc. for lookups and updates
  4. Process the map for the final result

‚è±Ô∏è Time Complexity

  • Insert/Search/Delete: O(1) average, O(n) worst-case
  • Iterate all entries: O(n)

üß™ Example

// Example: Two Sum
int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[0];
}
