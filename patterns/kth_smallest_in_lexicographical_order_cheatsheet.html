üìå What Is It?

The K-th Smallest in Lexicographical Order problem (LeetCode 440) asks you to find the k-th smallest integer in lexicographical order from 1 to n.

üéØ Solution Pattern (Prefix Counting / Trie Traversal)

// Java Solution (Prefix Counting)
class Solution {
    public int findKthNumber(int n, int k) {
        int curr = 1;
        k--;
        while (k > 0) {
            long steps = countSteps(n, curr, curr + 1);
            if (steps <= k) {
                curr++;
                k -= steps;
            } else {
                curr *= 10;
                k--;
            }
        }
        return curr;
    }

    private long countSteps(int n, long curr, long next) {
        long steps = 0;
        while (curr <= n) {
            steps += Math.min(n + 1, next) - curr;
            curr *= 10;
            next *= 10;
        }
        return steps;
    }
}

‚úÖ Use Cases

  • Lexicographical enumeration
  • Trie-based number traversal
  • Efficient k-th element search in large ranges

üìù Algorithm Steps

  1. Start from 1 (the smallest prefix)
  2. At each step, count how many numbers are under the current prefix
  3. If k is larger than the count, move to the next prefix (curr++)
  4. Otherwise, go deeper (curr *= 10) and decrease k
  5. Repeat until k becomes 0

‚è±Ô∏è Time Complexity

  • Time: O(log n * log n) (each prefix step and depth)
  • Space: O(1)

‚ö†Ô∏è Edge Cases to Consider

  • k = 1 (should return 1)
  • n is a single digit
  • k equals n
