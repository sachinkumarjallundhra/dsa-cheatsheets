üìå What is the Fast and Slow Pointer Pattern?

The Fast and Slow Pointer (also called Floyd‚Äôs Tortoise and Hare) is a powerful technique for solving linked list and array problems. It uses two pointers that move at different speeds‚Äîtypically, the "slow" pointer moves one step at a time, while the "fast" pointer moves two steps. This pattern helps you:

  • Detect cycles (loops) in data structures
  • Find the middle of a list
  • Identify duplicate elements
  • Solve problems with circular or repeating structures

Why is it so useful in interviews?
It lets you solve problems in O(N) time and O(1) space, which is often optimal and impresses interviewers!

üß± Pattern Template

ListNode slow = head;
ListNode fast = head;

while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}

  • Detect cycles in linked list or array
  • Find middle of a linked list
  • Check if a linked list is a palindrome
  • Detect infinite loop (e.g., Happy Number)
  • Find circular paths in arrays

üìò Common LeetCode Problems

üü¢ Easy Problems

  • Linked List Cycle Easy
    LeetCode Cheat Sheet
    Detect if a linked list has a cycle using fast and slow pointers.
    How it works:
    • Start both pointers at the head.
    • Move slow by 1 step, fast by 2 steps each iteration.
    • If fast meets slow, a cycle exists.
    • If fast reaches the end, no cycle exists.
  • Middle of the Linked List Easy
    LeetCode Cheat Sheet
    Find the middle node of a linked list.
    How it works:
    • Start both pointers at the head.
    • Move slow by 1 step, fast by 2 steps each iteration.
    • When fast reaches the end, slow will be at the middle node.
  • Happy Number Easy
    LeetCode Cheat Sheet
    Detect cycles in number transformation using fast and slow pointers.
    How it works:
    • Apply the transformation to both pointers: slow moves one step, fast moves two steps.
    • If fast meets slow, a cycle exists (not a happy number).
    • If fast reaches 1, the number is happy.

üü° Medium Problems

  • Palindrome Linked List Medium
    LeetCode Cheat Sheet
    Check if a linked list is a palindrome using fast and slow pointers.
    How it works:
    • Use fast/slow pointers to find the middle of the list.
    • Reverse the second half of the list.
    • Compare the first and second halves node by node.
    • If all nodes match, the list is a palindrome.
  • Circular Array Loop Medium
    LeetCode Cheat Sheet
    Detect cycles in a circular array using fast and slow pointers.
    How it works:
    • Use array indices as pointers.
    • Move slow by 1 step, fast by 2 steps, wrapping around the array.
    • If fast meets slow, a cycle exists.
    • Check direction and length to ensure a valid cycle.
  • Maximum Twin Sum of a Linked List Medium
    LeetCode Cheat Sheet
    Find the maximum twin sum in a linked list using fast and slow pointers and list reversal.
    How it works:
    • Use fast/slow pointers to find the middle of the list.
    • Reverse the second half of the list.
    • Pair nodes from the start and end, summing their values.
    • Track the maximum twin sum.
  • Split a Circular Linked List Medium
    LeetCode Cheat Sheet
    Split a circular linked list into two halves using fast and slow pointers.
    How it works:
    • Use fast/slow pointers to find the middle of the circular list.
    • Break the list into two halves at the middle and end.
    • Reconnect the ends to form two separate circular lists.

üî¥ Hard Problems

  • Find the Duplicate Number Hard
    LeetCode Cheat Sheet
    Find the duplicate in an array using fast and slow pointers (Floyd's Tortoise and Hare).
    How it works:
    • Treat array values as pointers (indices).
    • Use fast/slow pointers to detect a cycle (duplicate).
    • Once they meet, reset one pointer to start and move both by 1 step to find the duplicate.
  • Linked List Cycle II Hard
    LeetCode Cheat Sheet
    Find the node where the cycle begins in a linked list using fast and slow pointers.
    How it works:
    • Use fast/slow pointers to detect a cycle.
    • When they meet, reset one pointer to the head.
    • Move both pointers one step at a time; where they meet again is the cycle start.

üß™ Example 1: Detect Cycle in Linked List

public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}
