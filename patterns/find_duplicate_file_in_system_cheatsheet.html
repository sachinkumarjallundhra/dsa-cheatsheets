üìå What Is It?

The Find Duplicate File in System problem (LeetCode 609) asks you to find all groups of duplicate files in a file system. Each file is represented by a directory path and file content. Return lists of file paths that have duplicate content.

üéØ Solution Pattern (HashMap for Content Grouping)

// Java Solution
class Solution {
    public List<List<String>> findDuplicate(String[] paths) {
        Map<String, List<String>> map = new HashMap<>();
        for (String path : paths) {
            String[] parts = path.split(" ");
            String dir = parts[0];
            for (int i = 1; i < parts.length; i++) {
                String file = parts[i];
                int idx = file.indexOf('(');
                String name = file.substring(0, idx);
                String content = file.substring(idx + 1, file.length() - 1);
                String fullPath = dir + "/" + name;
                map.computeIfAbsent(content, k -> new ArrayList<>()).add(fullPath);
            }
        }
        List<List<String>> res = new ArrayList<>();
        for (List<String> group : map.values()) {
            if (group.size() > 1) res.add(group);
        }
        return res;
    }
}

‚úÖ Use Cases

  • Finding duplicate files by content
  • Data deduplication in storage systems
  • Grouping items by value or property

üìù Algorithm Steps

  1. Iterate through each path string
  2. For each file, extract the file name and content
  3. Use a HashMap to group file paths by their content
  4. Collect groups with more than one file path

‚è±Ô∏è Time Complexity

  • Time: O(N * L) where N is the number of files and L is the average length of each path
  • Space: O(N * L) for storing file paths in the map

‚ö†Ô∏è Edge Cases to Consider

  • No duplicate files (should return empty list)
  • Multiple files with the same content in different directories
  • Empty input array
