🔢 Unique Number of Occurrences – Java Cheat Sheet

📌 What Is It?

The Unique Number of Occurrences problem (LeetCode 1207) asks you to determine if the number of occurrences of each value in an array is unique.

🎯 Solution Pattern (HashMap + HashSet)

// Java Solution
class Solution {
    public boolean uniqueOccurrences(int[] arr) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int num : arr) freq.put(num, freq.getOrDefault(num, 0) + 1);
        Set<Integer> set = new HashSet<>(freq.values());
        return set.size() == freq.size();
    }
}

✅ Use Cases

  • Checking uniqueness of frequency counts
  • Interview questions on arrays and hash maps
  • Data validation and analytics

📝 Algorithm Steps

  1. Count the frequency of each element using a HashMap
  2. Add all frequencies to a HashSet
  3. If the size of the set equals the size of the map, all frequencies are unique

⏱️ Time Complexity

  • Time: O(N) where N is the length of the array
  • Space: O(N) for the map and set

⚠️ Edge Cases to Consider

  • All elements are the same
  • All elements are unique
  • Negative numbers and zeros

🧪 Example

Input: arr = [1,2,2,1,1,3]
Output: true
Explanation: 1 occurs 3 times, 2 occurs 2 times, 3 occurs 1 time. All frequencies are unique.

Input: arr = [1,2]
Output: false
Explanation: Both 1 and 2 occur once, so frequencies are not unique.
