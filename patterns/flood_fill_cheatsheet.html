üìå What Is It?

The Flood Fill problem involves changing the color of a region in a 2D image starting from a given pixel and replacing all connected pixels of the same color with a new color. It is commonly used in image editing and path-finding algorithms.

üß± Pattern Template

class Solution {
    public int[][] floodFill(int[][] image, int sr, int sc, int newColor) {
        int originalColor = image[sr][sc];
        if (originalColor != newColor) {
            dfs(image, sr, sc, originalColor, newColor);
        }
        return image;
    }

    private void dfs(int[][] image, int r, int c, int color, int newColor) {
        if (r < 0 || r >= image.length || c < 0 || c >= image[0].length || image[r][c] != color) {
            return;
        }
        image[r][c] = newColor;
        dfs(image, r + 1, c, color, newColor);
        dfs(image, r - 1, c, color, newColor);
        dfs(image, r, c + 1, color, newColor);
        dfs(image, r, c - 1, color, newColor);
    }
}

üìä Time Complexity

  • Time Complexity: O(N), where N is the number of pixels in the image. Each pixel is visited at most once.
  • Space Complexity: O(N), for the recursion stack in the worst case (all pixels are connected).

‚úÖ Use Cases

  • Image editing (paint bucket tool).
  • Connected component labeling in grids.
  • Path-finding and region-filling algorithms.

üß™ Example

Input: image = [[1,1,1],[1,1,0],[1,0,1]], sr = 1, sc = 1, newColor = 2
Output: [[2,2,2],[2,2,0],[2,0,1]]

Explanation:
- Starting from (1,1), all connected 1's are changed to 2.

üí° Pro Tips

  • Always check for the base case to avoid infinite recursion.
  • Use BFS for iterative solutions to avoid stack overflow on large images.
  • Edge cases: The new color is the same as the original color, or the image is empty.
