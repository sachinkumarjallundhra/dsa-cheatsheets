The Logger Rate Limiter problem (LeetCode 359) asks you to design a logger system that receives a stream of messages along with their timestamps, and returns true if the message should be printed (not printed in the last 10 seconds), otherwise returns false.

ğŸ¯ Solution Pattern (HashMap for Timestamp Tracking)

// Java Solution
class Logger {
    private Map<String, Integer> map;

    public Logger() {
        map = new HashMap<>();
    }

    public boolean shouldPrintMessage(int timestamp, String message) {
        if (!map.containsKey(message) || timestamp - map.get(message) >= 10) {
            map.put(message, timestamp);
            return true;
        }
        return false;
    }
}

ğŸ“ Algorithm Steps

  1. Initialize a HashMap to store the last printed timestamp for each message
  2. For each incoming message, check if it was printed in the last 10 seconds
  3. If not, update the timestamp and return true
  4. If yes, return false
