üìå What Is It?

Stack Pattern is used for problems involving Last-In-First-Out (LIFO) operations, tracking nested structures, and maintaining ordered state. Common applications include parsing expressions, validating brackets, and finding next greater/smaller elements.

üéØ Solution Patterns

1. Basic Stack Operations

Stack<Type> stack = new Stack<>();
stack.push(element);    // Add to top
stack.pop();           // Remove from top
stack.peek();         // View top element
stack.isEmpty();      // Check if empty

2. Monotonic Stack Pattern

Stack<Integer> stack = new Stack<>();
for (int i = 0; i < arr.length; i++) {
    // For next greater element (increasing stack)
    while (!stack.isEmpty() && arr[i] > arr[stack.peek()]) {
        int prevIndex = stack.pop();
        result[prevIndex] = arr[i];  // Found next greater element
    }
    stack.push(i);
}

3. Parentheses Validation Pattern

Stack<Character> stack = new Stack<>();
for (char c : s.toCharArray()) {
    if (c == '(' || c == '{' || c == '[') {
        stack.push(c);
    } else {
        if (stack.isEmpty()) return false;
        if (c == ')' && stack.peek() != '(') return false;
        if (c == '}' && stack.peek() != '{') return false;
        if (c == ']' && stack.peek() != '[') return false;
        stack.pop();
    }
}

üí° Key Tips

  • Always validate stack is not empty before pop/peek operations
  • Consider using multiple stacks for complex problems
  • For monotonic stack problems, decide whether to maintain increasing or decreasing order
  • Use StringBuilder for string manipulation problems
  • Consider edge cases: empty input, single element, all same elements
