🧮 Dynamic Programming – Java Cheat Sheet

📌 What Is It?

Dynamic Programming (DP) is a technique for solving problems by breaking them down into overlapping subproblems, solving each subproblem just once, and storing their solutions. DP is used to optimize recursive algorithms by avoiding redundant computations.

🧱 Pattern Template (Top-Down with Memoization)

Map<Key, Value> memo = new HashMap<>();

Value dp(Key key) {
    if (memo.containsKey(key)) return memo.get(key);
    // Base case
    if (isBaseCase(key)) return baseValue;
    // Recurrence
    Value result = ...; // compute using dp(subproblems)
    memo.put(key, result);
    return result;
}

🧱 Pattern Template (Bottom-Up Tabulation)

Value[][] dp = new Value[N][M]; // or int[]/int[][]/boolean[]/boolean[][] etc.

for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        if (isBaseCase(i, j)) {
            dp[i][j] = baseValue;
        } else {
            dp[i][j] = ...; // compute using dp[...]
        }
    }
}

  • Problems with optimal substructure and overlapping subproblems

  • Grid-based pathfinding, string matching, partitioning, and more

🧪 Example: Climbing Stairs

Input: n = 5
Output: 8

Explanation:
- dp[i] = dp[i-1] + dp[i-2]
- Base cases: dp[0] = 1, dp[1] = 1
- dp[2] = 2, dp[3] = 3, dp[4] = 5, dp[5] = 8

💡 Pro Tips

  • Identify overlapping subproblems and optimal substructure.
  • Use memoization (top-down) or tabulation (bottom-up) to avoid recomputation.
  • Optimize space by using 1D arrays when possible.
  • Edge cases: empty input, single-element input.
