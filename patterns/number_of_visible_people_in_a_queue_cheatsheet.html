👀 Number of Visible People in a Queue – Java Cheat Sheet

📌 What Is It?

The Number of Visible People in a Queue problem (LeetCode 1944) asks you, for each person in a queue (represented by their heights), to count how many people to their right they can see. A person can see another person if everyone in between is shorter, and the first person taller or equal to them blocks the view.

🧱 Pattern Template (Monotonic Stack)

class Solution {
    public int[] canSeePersonsCount(int[] heights) {
        int n = heights.length;
        int[] res = new int[n];
        Stack<Integer> stack = new Stack<>();
        for (int i = n - 1; i >= 0; i--) {
            int count = 0;
            while (!stack.isEmpty() && heights[i] > stack.peek()) {
                stack.pop();
                count++;
            }
            if (!stack.isEmpty()) count++;
            stack.push(heights[i]);
            res[i] = count;
        }
        return res;
    }
}

✅ Use Cases

  • Monotonic stack for "next greater" or "visible" problems
  • Interval and visibility counting
  • Stack-based array traversal

💡 Pro Tips

  • Process from right to left using a stack to track visible people.
  • Pop all shorter people and count them; the first taller or equal person is also counted.
